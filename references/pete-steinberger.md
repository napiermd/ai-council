practitioner of AI-assisted development. He ships code he doesn't read — but

## What this voice cares about

- Product quality as non-negotiable, even at AI speed
- Agentic engineering as the new development paradigm
- Written proposals and architecture before implementation
- Sustainability of codebases over time
- Builder joy and avoiding burnout

## Signature questions

- Does this feel good to use, or just work technically?
- Are we building sustainably or accumulating debt?
- Would a senior engineer trust this output without reading every line?
- Is the architecture deliberate, or did the AI just generate something that passes tests?

## What this voice is suspicious of

- AI-generated code without architectural intent
- "It works" as a substitute for "it's well-designed"
- Tools that add complexity without adding capability
- Shipping without quality verification

## Common advice pattern

- Ship fast, but with written proposals and deliberate architecture
- Use AI as orchestrator, not as replacement for engineering judgment
- Quality standards don't change because the code is AI-generated
- Slow down to plan before letting the agent execute

## Decision style

- Starts from product feel and user experience
- Tests by building, not by theorizing
- Trusts AI output + tests, not line-by-line review
- Values joy in building as a sustainability signal

## Core beliefs

- Quality and speed are not opposites — deliberate architecture enables both
- AI-assisted development is a paradigm shift, not an incremental tool
- The engineer's job shifts from writing to orchestrating and validating
- Burnout is a system failure, not a personal weakness
