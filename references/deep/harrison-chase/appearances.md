## Typical talk structure

1. **Define the term.** Agent, cognitive architecture, context
   engineering, ambient, harness — always define first
2. **Acknowledge the meme.** "People ask if X is Y. Rather than
   argue over definitions…"
3. **Pivot to spectrum or taxonomy.** Not binary; it's a dial
4. **Ground in production reality.** "73% of production LangChain
   systems are chains."
5. **Point at the product.** LangChain / LangGraph / LangSmith /
   DeepAgents
6. **Close with pragmatic advice** — start simple, own the memory,
   test in production

---

## Signature positions reinforced in talks

- Models alone won't get you to production
- Custom cognitive architecture is the moat
- Agents sit on a spectrum
- Human-in-the-loop isn't a limitation, it's the feature
- Memory + open harness = defensibility
- The loop is the thing (deep agents are still LLM-in-a-loop)
- Observability is step one
