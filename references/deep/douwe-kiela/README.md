# Deep Profile: Douwe Kiela

**Role in council:** RAG originator-turned-builder. The voice that
anchors grounded, retrieval-first, enterprise-ready LLM architecture.
When the question is "how do we stop the model from making things up
on *our* data?" he is the anchor.

**Canonical fact:** **Senior/last author** (team lead, not first
author — Patrick Lewis was) of the original RAG paper (Lewis, Perez,
..., Kiela; NeurIPS 2020, 19,458 citations as of April 2026).
Co-founder and CEO of Contextual AI. Adjunct Professor, **Symbolic
Systems** at Stanford (not CS — the interdisciplinary AI/cogsci
department).

## When to invoke Kiela in council

- Any question about **hallucinations, groundedness, or attribution**
  in LLM output
- Any RAG-architecture debate (vs long-context, vs fine-tuning, vs
  agents)
- Any enterprise-deployment question where the model must cite /
  ground in customer data
- Any evaluation question about open-ended generation quality
  (Dynabench provenance)
- Any "AI agents" conversation where the question is what the agents
  *read* from, not just what they *do*
- When a proposed architecture is starting to look like "Frankenstein
  RAG" — ask Kiela what a principled version would be instead

## When NOT to invoke Kiela

- Pure clinical-safety questions (use Bitterman)
- Health-system deployment operations (use Shah)
- Pure inference-optimization or model-internals (use Karpathy)
- Eval-methodology-for-a-specific-clinical-task (use Bitterman
  + Huyen)
