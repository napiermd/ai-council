# Profile Standard

Every council profile should be source-backed, not purely vibes-based.

Pair this file with:

- [`source-index.md`](./source-index.md)
- [`refresh-checklist.md`](./refresh-checklist.md)
- [`../signals/README.md`](../signals/README.md)

## Required sections

Each `references/*.md` voice file should include:

1. `Core domain`
2. `Why this voice matters in this council`
3. `Canonical sources`
4. `Current signals`
5. `What this voice cares about`
6. `Signature questions`
7. `What this voice respects`
8. `What this voice is suspicious of`
9. `What this voice would push on`
10. `Common advice pattern`
11. `Best paired with`
12. `Tension with other voices`
13. `Last reviewed`

## Recommended enhanced sections

For higher-fidelity voices, also add:

14. `Decision style`
15. `Core beliefs` or `Worldview`
16. `Likely next move (inference)`
17. `Watch sources`

These are interpretive sections and should be clearly written as distillation or
inference, not as direct biography.

## Source rules

- Prefer official websites, official bios, official lab pages, official GitHub profiles, and first-party writing.
- Use secondary news only when it adds something genuinely current that is not visible in official sources.
- Do not pretend the profile reflects a person's full body of work unless the sources actually support that.
- Distill patterns, not biographies.

## Current signals

`Current signals` should answer:

- what they are working on now
- what role they currently hold
- what themes are currently prominent in their public output

Current signals should be lightweight and source-cited.

When recency matters, pair the hand-authored profile with the generated file in
`signals/recent/<voice>.md`.

## Last reviewed

Every profile should end with:

- review date
- source count
- whether the profile is fully source-backed or partially source-backed

Every refresh should also update [`source-index.md`](./source-index.md).
